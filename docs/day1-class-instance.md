### 자바에서 대입(=)은 항상 변수에 들어 있는 값을 복사한다.

→ 기본형(primitive)은 실제 값을 복사하고 참조형(reference)은 객체 주소(참조값)를 복사

→ 참조값도 값이다!

- **클래스**
    - 사용자가 직접 타입을 정의하는 것, 그걸 위한 설계도 즉, 붕어빵 틀임.
    - 클래스로 정의한 것은 설계도일 뿐, 실제 메모리에 올라가는 건 인스턴스를 만들 때임.
    - 학생을 관리할 때 이름은 이름끼리, 학년은 학년끼리, 점수는 점수끼리 관리한다면 헷갈리고 사람이 관리하기 어려움 → 그래서 한 학생의 이름, 학년, 점수를 한번에 관리하는 것이 좋음. 이러기 위해서 학생 이라는 클래스를 만드는 것. 학생이라는 클래스 안에는 이름, 학년, 점수 라는 멤버변수(=필드)가 있는 것임.
- **인스턴스(=객체)**
    - 클래스를 사용해서 실제 메모리에 만들어진 실체
    - new 키워드를 통해 만들어짐
    - 학생이라는 클래스 안에서 만들어진 학생1, 학생2, 학생3 등등이 모두 객체임.
    - 인스턴스와 객체는 같은 의미이지만 뉘앙스가 다름. 보통 student 클래스에서 만들어진 student1 인스턴스. 이렇게 말함.

- 향상된 for 문
    - 원래 for문

    ```java
    for(int i = 0; i < students.length; i++) {
    	Student s = students[i];
    	System.out.println("이름: " + s.name + ", 나이: " + s.age + ...);
    }
    ```

    - 향상된 for문

    ```java
    for(Student s : students) {
    	System.out.println("이름: " + s.name + ", 나이: " + s.age + ...);
    }
    ```

  즉,

    ```java
    for(int i = 0; i < students.length; i++) {
    	Student s = students[i];        //이 두 문장이
    
    -> for(Student s : students) {    //이렇게 바뀌었다
    //문법
    for(타입 변수명: 배열 혹은 컬렉션) { 
    	//반복할 코드
    }
    ```

  → 향상된 for문의 단축키 : `iter`

    - 향상된 for문은 편하지만 **인덱스가 필요할 때는 사용하지 못한다**는 단점이 있음.

      (예: 배열 중 특정 값 찾고 그 뒤 값 바꾸고 싶을 때 등)
