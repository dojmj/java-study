### 키워드: 모듈화(절차지향), 캡슐화(객체지향)

- 절차 지향 프로그래밍: 순서에 맞게 코딩. **“어떻게”**에 초점을 맞춤
    - **모듈화**: 각각의 기능을 메서드로 추출하는 것. 레고처럼 필요한 기능을 호출해서 쓰거나, 그 부분만 수정가능
    - 한계: 데이터와 기능(메서드)이 분리되어 있음. 데이터와 그 데이터를 사용하는 기능은 밀접한 연관이 있으나, 다른 파일에서 관리하기 때문에 유지보수 관점에서 효율적이지 않음. → 이걸 해결한게 객체 지향 프로그래밍
- 객체 지향 프로그래밍: 객체를 중요하게 생각. **“무엇을”**에 초점을 맞춤
    - 클래스에는 속성(데이터, 필드, 멤버변수) 뿐만 아니라 기능(메서드)도 함께 포함할 수 있음.

        ```java
        //지금까지는 아래와 같이 클래스를 정의했다면,
        public class ValueData {
        	int value;
        }
        
        //멤버 변수 뿐만 아니라 아래와 같이 메서드도 함께 포함 가능함.
        public class ValueObject {
        	int value;
        	
        	void add() { //단, 여기서 만드는 add() 메서드에는 static 키워드 사용안함
        		value++;
        		System.out.println("숫자 증가 value=" + value);
        	}
        }
        ```

    - 이렇게 클래스에 속성과 기능을 함께 포함하면 데이터와 기능을 한 곳에서 관리할 수 있음
    - 인스턴스(객체)의 메서드를 호출할 때도 멤버 변수 호출과 동일하게 `.(dot)` 으로 접근하여 호출

      → 객체는 자신의 메서드를 통해 자신의 멤버 변수에 접근 가능

      → 객체의 메서드 내부에서 접근하는 멤버 변수는 개체 자신의 멤버 변수이다.

    - **캡슐화**: 속성과 기능을 하나의 단위(클래스)로 묶어서 필요한 기능을 메서드를 통해 외부에 제공하는 것(내부 구현을 외부에 숨김)
        - 즉, 하나의 클래스에 멤버 변수와 메서드를 정의해서 캡슐처럼 묶어서 외부에 제공하는 것
